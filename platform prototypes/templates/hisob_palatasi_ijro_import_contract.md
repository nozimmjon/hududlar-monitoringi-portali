# Ҳисоб палатаси ижро текшируви: platform import contract

Ушбу contract `hisob_palatasi_ijro_tekshiruv_template.xlsx` файлидан platformнинг `Ижро мониторинги` бўлимига маълумот қабул қилиш қоидасини белгилайди.

## Асосий workflow

1. Ҳисоб палатаси `02_Текширув киритиш`, `03_Ойлик текширув` ёки `04_Туман кесими` варағига текширув натижасини киритади.
2. `Платформага қабул` устуни `Тайёр` бўлса, қатор platformга импорт қилинади.
3. Import қилинган қатор `Ижро мониторинги` журналида кўринади.
4. Фақат `Ҳисобот ҳолати = Тасдиқланди` бўлган қатор KPI `амалда` қийматига қўшилади.
5. `Қайтарилди`, `Рад этилди`, `Кўриб чиқилмоқда` ҳолатлари KPIга қўшилмайди, лекин ижро/муаммо мониторингида қолади.

## Import key

| Варақ | Import key | Platform объекти |
|---|---|---|
| `02_Текширув киритиш` | `T-ID + Текширув даври + Ҳудуд` | Топшириқ бўйича умумий текширув |
| `03_Ойлик текширув` | `M-ID + T-ID + Ҳисобот ойи + Ҳудуд` | Ойлик текширув |
| `04_Туман кесими` | `D-ID + T-ID + Туман/шаҳар + Текширув даври` | Туман кесимидаги текширув |

Эслатма: ҳозирги `00_Чора_тадбир_Андижон.docx` реестрида алоҳида ойлик M-қаторлар ажратилмаган. `03_Ойлик текширув` варағи кейинги босқичда ойлик мониторинг талаби пайдо бўлса ишлатилади.

## Field mapping

| Excel майдони | Platform field | Изоҳ |
|---|---|---|
| `T-ID` | `taskId` | `T-001...T-054` чора-тадбир реестри билан боғланади |
| `M-ID` | `monthlyCheckId` | Фақат ойлик варақда |
| `D-ID` | `districtCheckId` | Фақат туман кесими варағида; ҳозирги Андижон template’ида `D-01...D-22` |
| `Source №` | `sourceNo` | Кафолат хатидаги қатор рақами |
| `Йўналиш` | `moduleLabel` | 7 жадвал/йўналиш мантиғи |
| `KPI боғланиши` | `kpi` | KPI code: `industry`, `budget`, `poverty` ва ҳ.к. |
| `Топшириқ матни` | `taskTitle` | Platformда task drawer ва журналда кўринади |
| `Текширув даври` | `period` | `q1`, `h1`, `m9`, `year`, ёки ойлик контекст |
| `Ҳисобот ойи` | `reportMonth` | Ойлик текширув учун мажбурий |
| `Ҳудуд` / `Туман/шаҳар` | `district` | Вилоят ёки аниқ туман/шаҳар |
| `Ҳисобот ҳолати` | `status` | `Киритилди`, `Кўриб чиқилмоқда`, `Тасдиқланди`, `Қайтарилди`, `Рад этилди` |
| `Ижро ҳолати` | `executionStatus` | `Бажарилди`, `Қисман бажарилди`, `Бажарилмади`, `Муддати кечикди`, `Маълумот йўқ` |
| `Далил ҳолати` | `evidenceStatus` | Далил сифати |
| `Тасдиқланган факт` | `actualValue` | KPIга қўшиладиган факт, агар status тасдиқланган бўлса |
| `Ўлчов бирлиги` | `unit` | `%`, `млрд сўм`, `млн $`, `та` ва ҳ.к. |
| `Муаммо тури` | `issueType` | Қайтарилган/кечиккан/далил етарсиз қаторларда мажбурий |
| `Ҳисоб палатаси изоҳи` | `comment` | Аудит изоҳи |
| `Тавсия / кўрсатма` | `recommendation` | Тузатиш бўйича кўрсатма |
| `Тузатиш муддати` | `correctionDeadline` | Correction queue учун |
| `Текширув санаси` | `checkedAt` | Platform журнал санаси |
| `Текширувчи` | `checkedBy` | Ҳисоб палатаси масъул ходими/гуруҳи |
| `Файл/ҳавола` | `evidenceName` | Далил файли ёки ҳавола |
| `Платформага қабул` | `importReadiness` | `Тайёр` бўлмаса импорт қилинмайди |

## Status mapping

| Excel `Ҳисобот ҳолати` | Platform status | KPIга таъсири |
|---|---|---|
| `Киритилди` | `submitted` | KPIга қўшилмайди |
| `Кўриб чиқилмоқда` | `review` | KPIга қўшилмайди |
| `Тасдиқланди` | `approved` | KPI `амалда` қийматига қўшилади |
| `Қайтарилди` | `rejected` | KPIга қўшилмайди, correction queueга тушади |
| `Рад этилди` | `rejected` | KPIга қўшилмайди, correction queueга тушади |

## Test scenarios

`hisob_palatasi_ijro_tekshiruv_test_filled.xlsx` файлда фақат platform import логикасини текшириш учун `НАМУНА` қаторлари бор. Бу файл реал Ҳисоб палатаси маълумоти ҳисобланмайди.

| Scenario | Sheet | ID | Нима текширилади |
|---|---|---|---|

| Returned task | `02` | `T-031` | Далил етарсиз бўлганда correction fields мажбурийлиги |
| District accepted task | `04` | `D-01 / T-048` | Туман кесимида тасдиқланган факт |
